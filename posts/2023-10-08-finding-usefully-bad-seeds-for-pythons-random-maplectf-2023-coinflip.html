At face value, it looks like the goal is Random.getrandbits to return a lot more zero bits than one bits, where ‚Äúa lot more‚Äù is a net bias of 7 million over 7‚Äì20 million attempts. That‚Äôs not going to happen by chance even with the brutest of brute force, but the ability to provide a random seed is pretty suggestive. Okay, progress! Looks like Python takes an arbitrarily large int and uses it pretty directly to seed a Mersenne Twister RNG ‚Äì no extra hashing or anything. Wikipedia has this to say about the algorithm:

  • Contains subsequences with more 0's than 1's. This adds to the poor diffusion property to make recovery from many-zero states difficult.

N = 624
M = 397
MATRIX_A = 0x9908b0df
UPPER_MASK = 0x8000_0000
LOWER_MASK = 0x7fff_ffff


class MersenneTwisterRng:
	def __init__(self):
		self.state = [0] * N
		self.index = 0

	def init_genrand(self, s):
		assert 0 <= s <= 0xffff_ffff

		mt = self.state = [0] * N
		mt[0] = s

		for i in range(1, N):
			mt[i] = (1812433253 * (mt[i-1] ^ (mt[i-1] >> 30)) + i) & 0xffff_ffff

		self.index = N

	def init_by_array(self, init_key):
		self.init_genrand(19650218)
		mt = self.state

		i = 1
		j = 0

		for _ in range(max(N, len(init_key))):
			mt[i] = ((mt[i] ^ ((mt[i-1] ^ (mt[i-1] >> 30)) * 1664525)) + init_key[j] + j) & 0xffff_ffff
			i += 1
			j += 1
			if i >= N:
				mt[0] = mt[N-1]
				i = 1
			if j >= len(init_key):
				j = 0

		for _ in range(N - 1):
			mt[i] = ((mt[i] ^ ((mt[i-1] ^ (mt[i-1] >> 30)) * 1566083941)) - i) & 0xffff_ffff
			i += 1
			if i >= N:
				mt[0] = mt[N-1]
				i = 1

		mt[0] = 0x8000_0000  # MSB is 1; assuring non-zero initial array

	def genrand_uint32(self):
		mt = self.state

		if self.index >= N:
			mag01 = (0, MATRIX_A)
			kk = 0

			while kk < N-M:
				y = (mt[kk]&UPPER_MASK)|(mt[kk+1]&LOWER_MASK)
				mt[kk] = mt[kk+M] ^ (y >> 1) ^ mag01[y & 0x1]
				kk += 1

			while kk < N - 1:
				y = (mt[kk]&UPPER_MASK)|(mt[kk+1]&LOWER_MASK)
				mt[kk] = mt[kk+(M-N)] ^ (y >> 1) ^ mag01[y & 0x1]
				kk += 1

			y = (mt[N-1]&UPPER_MASK)|(mt[0]&LOWER_MASK)
			mt[N-1] = mt[M-1] ^ (y >> 1) ^ mag01[y & 0x1]

			self.index = 0

		y = mt[self.index]
		self.index += 1
		y ^= y >> 11
		y ^= (y << 7) & 0x9d2c5680
		y ^= (y << 15) & 0xefc60000
		y ^= y >> 18
		return y

Looking at that mixing in genrand_uint32, I can see how it would propagate zero-heaviness in a state: the only operation that can bring in new 1 bits is XORing with mag01[y & 0x1], and mag01[0] is still 0! If init_by_array didn‚Äôt guarantee at least one set bit, an initially zero state would stay zero forever. But even though at least this bit is going to be set no matter what, taking into consideration that little bullet point from Wikipedia, what if we artificially try getting as many zeros as possible?

Let‚Äôs patch the seeding function:

That‚Äôs a big deal! If we can get the Random instance to start in this state, we can capture the flag. But is it actually possible to produce a state this good from a seed? Well, let‚Äôs give it a try, working backwards one step at a time.

This step again has the convenient property that the value of mt[i] after an iteration is a simple sum of

  • a function of mt[i-1] and mt[i],
  • j,
  • and a word from init_key (the seed).

Setting mt[i] to an arbitrary value is as easy as calculating the stuff that gets added to init_key[j] and cancelling it out with the magic of the additive inverse before adding the desired value.

for _ in range(max(N, len(init_key))):
	desired = i
	stuff = (mt[i] ^ ((mt[i-1] ^ (mt[i-1] >> 30)) * 1664525)) + j
	init_key[j] = (desired - stuff) & 0xffff_ffff

We can make init_key the right length to set i to any value we want within range, too, since it increments through the 1..N−1 range according to according to max(N, len(init_key)).

j = 0

	init_key[j] = (desired - stuff) & 0xffff_ffff
